The Four Rules of Linked Data

So in a way, Linked Data is the Semantic Web realized via four best practice principles.

  1. Use URIs as names for things. An example of a URI is any URL. For example: https://cambridgesemantics.com/blog/semantic-university/ is the URI for Semantic University.
  2. Use HTTP URIs so that people can look up those names.
  3. When someone looks up a URI, provide useful information, using the standards such as RDF* and SPARQL.
  4. Include links to other URIs so that they can discover more things.
