Copy and Move Rules in C++

(Acknowledgements to http://www.modernescpp.com/index.php/c-core-guidelines-copy-and-move-rules/, by Rainer Grimm)

Let' begin with a meta-rule:

Value types, which behave like an int, should be copyable, but interfaces in class hierarchies are not.

Here are the eight rules:

  1. Make copy assignment non-virtual, take the parameter by const&, and return by non-const& (you should not change the source of your assignment)
  2. A copy operation should copy. (Copying can be a deep or shallow.)
  3. Make copy assignment safe for self-assignment
  4. Make move assignment non-virtual, take the parameter by &&, and return by non-const&
  5. A move operation should move and leave its source in a valid state
  6. Make move assignment safe for self-assignment. (No copying if this==src.this.)
  7. Make move operations noexcept
  8. A base class should suppress copying, and provide a virtual clone instead, if copying is desired
